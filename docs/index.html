Memory writes need an admission path

Most agent-memory tooling focuses on storing and retrieving more context. WriteFence focuses on the write path: what should be allowed to become memory in the first place.

Reject bad writes

Malformed, policy-breaking, or sensitive memory is stopped before it reaches long-term state.

Review uncertain writes

Review-required memory stays local until an operator approves, rejects, or merges it with existing context.

Replay policy changes

The WAL shows how prior writes would evaluate under the current rule set before policy changes land.

Write admission, not storage

WriteFence is not a vector database, retrieval layer, hosted service, or autonomous memory editor. It is a small control point before persistence.

The alpha is intentionally narrow: local HTTP write admission, audit logs, quarantine review, replay, and operator tooling.

Admission pipeline Inspect before persistence
Attempt An agent, MCP client, or app sends a memory write to the local proxy.
Evaluate WriteFence applies deterministic local rules and optional semantic checks before persistence.
Decision outcomes
Allow or warn Safe writes move upstream; warnings keep operator review visible.
Quarantine Review-required writes stay local until an operator approves or rejects them.
Block Policy-breaking writes return a structured admission decision.

Three-minute local preview

Run the alpha with the included mock memory store. No LightRAG, Qdrant, Ollama, auth, billing, or hosted service is required for the default preview.

go build -o bin/writefence ./cmd/writefence
go build -o bin/writefence-cli ./cmd/writefence-cli
go run ./demo/mock-memory-store.go -addr 127.0.0.1:9621

WRITEFENCE_DATA_DIR=/tmp/writefence-alpha \
  ./bin/writefence --addr 127.0.0.1:9622 --upstream http://127.0.0.1:9621

WRITEFENCE_DATA_DIR=/tmp/writefence-alpha \
  WRITEFENCE_WAL=/tmp/writefence-alpha/writefence-wal.jsonl \
  ./demo/canonical-demo.sh
Open http://127.0.0.1:9622/_writefence for the local operator UI. Semantic quarantine requires embeddings and Qdrant; without them, the demo prints a note and continues with deterministic local rules.

Operator views

The local UI is built for repeated operator work: inspect decisions, review suggested fixes, replay the WAL, and confirm runtime configuration.

Overview shows live admission state, recent decisions, and active rules.

Alpha scope

Included now

  • HTTP reverse proxy for document writes.
  • Four-state admission decisions: allowed, warned, quarantined, blocked.
  • Local WAL, violations log, quarantine log, replay, CLI, MCP server, and operator UI.

Not in this alpha

  • Hosted SaaS, multi-tenant auth, billing, or compliance claims.
  • A custom policy DSL or broad adapter matrix for every memory backend.
  • A memory database or retrieval engine.
